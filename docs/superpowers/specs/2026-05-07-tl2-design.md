# TL2 Design Spec - 2026-05-07

## Overview

TL2 = Upload maps pipeline. Chạy sau TL1 trong Main.ps1 menu. Repo download từ GitHub zip, extract TEMP, chạy Python embed.

## Architecture

```
TEMP/AutomationMinecraftInternetBar-main/ (repo extracted)
├── Main.ps1
├── python_embed/ (Python embed, shared all TL2)
├── TL2/
│   ├── lib_pure/
│   │   └── aes_pure.py (pure Python AES256)
│   ├── Decode.py
│   ├── Shield.py
│   ├── Menu2.py
│   ├── Manager.ps1
│   ├── nothing.enc (tải từ GitHub raw)
│   ├── Code.txt (output sau decrypt)
│   └── Stage_upload/
│       ├── Upload.py
│       └── Manager.py
```

## Flow

### 1. Main.ps1 (TL2 trigger)

- User chạy `irm https://.../Main.ps1 | iex`
- Tải repo zip từ `https://github.com/Lynstria/AutomationMinecraftInternetBar/archive/refs/heads/main.zip`
- Extract vào `TEMP/AutomationMinecraftInternetBar-main/`
- Tải `nothing.enc` từ GitHub raw vào `TL2/` folder
- Tải Python embed zip, extract vào `TEMP/python_embed/`
- User chọn option 2 -> chạy `python.exe TL2/Decode.py`

### 2. Decode.py

- User nhập password qua `input()`
- `aes_pure.py` decrypt `nothing.enc` với `SHA256(password)` làm key
- Parse JSON result, ghi full JSON ra `Code.txt` (cùng folder)
- Test Discord webhook URL (field `discord`) bằng `urllib.request.urlopen()` HEAD request
- Valid -> exit 0, tiếp tục Shield.py
- Invalid -> print lỗi, nhập lại (max 4 lần)
- Hết 4 lần -> exit 1, quay lại Main.ps1 menu

### 3. Shield.py

- Đọc `Code.txt`, lấy Discord webhook URL (field `discord`)
- Sinh random 4-digit (0000-9999) bằng `random.randint()`
- Đảo ngẫu nhiên thứ tự các chữ số trước khi gửi
- Gửi qua Discord webhook format: `Mã xác thực: <4-digit-đảo>`
- User xem Discord trên điện thoại, nhập 4-digit theo thứ tự hiển thị (đã đảo)
- So sánh với mã đã đảo (không đảo ngược lại)
- Sai -> nhập lại (max 4 lần)
- Hết 4 lần -> exit 1, quay lại Main.ps1
- Đúng -> exit 0, tiếp tục Menu2.py

### 4. Menu2.py

- Hiện menu: `1. Upload` / `2. Manager`
- Dùng `input()` chọn
- Chọn 1 -> chạy `Stage_upload/Upload.py`
- Chọn 2 -> chạy `Stage_upload/Manager.py`
- Sau khi Upload.py hoặc Manager.py xong -> quay lại Menu2.py (loop)
- Chọn khác -> nhập lại

### 5. Upload.py

- Truy cập `%APPDATA%\.minecraft\versions\`
- Nén entire `versions/` folder thành `versions-YYYY-MM-DD.zip` (store method, không compress)
- Đọc `Code.txt`, lấy Drive API credentials (field `drive`)
- Upload file zip lên Drive folder "Minecraft_Map"
- Drive folder link: `https://drive.google.com/drive/folders/1SJYI54NEQXb7a7OfZODSEO96rAUvPN12`
- Sau khi upload xong -> exit 0, quay lại Menu2.py

### 6. Manager.py + Manager.ps1

- Manager.py:
  - Đọc Drive credentials từ `Code.txt` (field `drive`)
  - Gọi Drive REST API list files trong "Minecraft_Map" folder
  - Ghi danh sách file ra temp JSON (ví dụ `TEMP/mc_versions.json`)
- Manager.ps1 (PowerShell UI):
  - Đọc temp JSON
  - Hiện list versions dạng hàng dọc
  - User dùng `Space` để toggle select (chỉ chọn được 1 phiên bản)
  - Option 1: Đặt làm bản chính (set default trên Drive để TL1 tải về)
  - Option 2: Xoá vĩnh viễn các phiên bản đã toggle
  - Option khác: Quay lại Menu2.py

## Key Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| AES impl | `aes_pure.py` (pure Python) | No pip, stdlib-compatible |
| nothing.enc | User tự encrypt ngoài | Decrypt only, no encrypt code |
| Discord mã format | `Mã xác thực: 1234` | Clear on mobile |
| Shield timeout | Không có (option 2) | KISS |
| Manager UI | PowerShell (.ps1) | User yêu cầu |
| Manager data pass | Temp JSON file | Simple, no arg escaping |
| Drive folder | "Minecraft_Map" | Fixed, link provided |
| Upload zip method | Store (no compression) | Faster, versions files already compressed |
| Retry logic | Decode.py 4 lần, Shield.py 4 lần | Fail early |
| Loop | Menu2.py loop until user exit | Not back to Main.ps1 |

## Constraints

- Python embed: no pip, stdlib only (trừ `aes_pure.py` tự code)
- Quán net: reset mỗi lần tắt -> không lưu config,一切在 TEMP
- Không động vào TL1 code
- Chỉ sửa Main.ps1 và code TL2
